Brent Welch's Practical Programming in Tcl and Tk is a good reference book. The 3rd edition is from 1999, and is still quite relevant, especially when augmented with the web references above. The 4th edition is coming soon.

Tcl's syntax is simpler and different from that of computer languages. It is based on only 11 rules. This simplicity makes Tcl behave in ways that are surprising to programmers used to languages like C or Perl. Tcl has

  • # for comments (only where Tcl expects a command name)
  • { for lists (and in effect, for command block structure)
  • $ for variable name substitution (not dereferencing)
  • () for arrays (associative, like Perl hashes)
  • command names (none of which are reserved)
None of them behaves exactly the way you might be used to. Eventually, you will realize that it's all good and consistent, but you can't assume that Tcl works like C or Perl or shell. Understanding the 11 syntax rules is the key to understanding Tcl.
